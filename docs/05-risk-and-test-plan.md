# 风险与测试计划

## 1. 主要风险

1. 沙盒不兼容：
  - 切换软链接后应用无法正常运行
2. 数据损坏：
  - 迁移期间数据库/日志仍在变化
3. 磁盘可用性：
  - 目标外接盘断开或变只读
4. 元数据漂移：
  - 记录状态与真实文件系统状态不一致

## 2. 对应缓解措施

1. 强制“应用已退出”预检
2. 先复制再校验，校验通过后再切换
3. 后检通过前不删除备份目录
4. 定期对账任务校验元数据与文件系统一致性
5. 按兼容性分级限制默认可迁移范围

## 3. 测试矩阵

| 类别 | 用例 | 期望结果 |
|---|---|---|
| 功能 | 迁移支持画像 | 成功并进入 `healthy` |
| 功能 | 成功迁移后执行回滚 | 原路径恢复可用 |
| 失败 | 复制阶段强制杀进程 | 源数据不丢失，可恢复 |
| 失败 | 软链接创建失败 | 自动回滚成功 |
| 失败 | 迁移后拔盘 | 出现 `degraded` 告警 |
| 兼容 | 尝试迁移 MAS 沙盒应用 | 被拦截或明确实验性提示 |
| 权限 | 缺少 Full Disk Access | 阻止操作并给出指引 |

## 4. 人工验证清单

1. 安装并准备应用测试画像
2. 构造大体积样本数据（建议 >2GB）
3. 执行迁移到外接盘
4. 重启应用并验证读写
5. 拔出外接盘并确认告警状态
6. 重新挂载并确认健康恢复
7. 执行回滚并验证应用恢复

## 5. 发布前检查

1. 画像分级标签已评审并可追踪
2. 迁移日志具备步骤级错误码
3. 回滚操作手册已成文
4. 已发布“已知限制”说明页
