# FAQ（V1）

日期：2026-03-05

## 1. 为什么我会被预检拦截？

常见原因包括：应用进程未退出、目标盘未挂载、目标盘只读、空间不足、权限不足。  
请根据错误码提示先修复环境，再重试迁移。

## 2. `bootstrap` 和 `migrate` 有什么区别？

- `bootstrap`：源路径无历史数据时，直接在外接盘初始化并建软链接。
- `migrate`：源路径已有数据时，执行复制/校验/切换/后检的事务流程。

## 3. 出现 `degraded` 一定要立刻回滚吗？

不一定。  
若是离线或只读导致，可先恢复外接盘状态后重新健康检查；  
若长时间无法恢复或出现 `critical` 对账项，建议执行回滚。

## 4. 对账任务会修改我的数据吗？

会执行“无损自动纠偏”，但不会搬运业务数据。  
当前默认会运行 `apply_safe_fixes=true`，仅处理安全动作（如清理残留 temp、同步元数据状态）。

## 5. 为什么仍会有“人工处理”提示？

V1 的自动修复只覆盖可证明无损的动作。  
涉及数据来源不明确或多路径冲突时，为防止误操作，必须人工确认。

## 6. 如何导出问题定位日志？

在“操作记录”页面查看失败项并展开关键步骤日志；仍无法定位时，可提供失败时间点和应用名称用于进一步排查。

## 7. 可以跨平台使用吗？

V1 仅面向 macOS，其他平台不在支持范围。
