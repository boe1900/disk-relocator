# 回滚操作手册（V1）

日期：2026-03-05

## 适用场景

1. 迁移后应用无法正常读写。
2. 健康面板出现 `broken` 或持续 `degraded`。
3. 对账任务出现 `critical` 漂移项。

## 标准回滚步骤

1. 关闭目标应用进程。
2. 在健康面板或迁移状态页执行“一键回滚”。
3. 等待回滚完成并确认状态为 `ROLLED_BACK`。
4. 执行一次健康检查，确认状态可恢复为 `healthy`。

## 回滚后检查清单

1. 源路径存在且不是软链接。
2. 应用可从源路径正常启动并读写。
3. 临时路径（`*.tmp.*`）已清理。
4. 操作日志中存在 `rollback` 阶段记录。

## 异常分支

1. `ROLLBACK_REMOVE_SYMLINK_FAILED`
   - 检查文件占用，关闭相关进程后重试。
2. `ROLLBACK_RESTORE_BACKUP_FAILED`
   - 进入人工恢复：核对 `.bak`、目标路径、源路径三者数据。
3. `ROLLBACK_CLEANUP_TEMP_FAILED`
   - 回滚已完成时可先保留，后续手动清理临时目录。
4. `ROLLBACK_FAILED`
   - 禁止再次迁移，先完成人工修复并运行对账。

## 人工恢复最低动作

1. 确认最终保留路径（通常为源路径）。
2. 删除错误软链接（若存在）。
3. 将 `.bak` 或目标数据恢复到源路径。
4. 重新执行健康检查与对账，确保状态一致。
