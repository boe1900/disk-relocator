# Disk Relocator Roadmap

## 目标

在 macOS 上提供“安全可控”的应用数据外接盘迁移能力，V1 聚焦以下 4 条主线：
1. 首次引导模式（首次直接落盘外接盘）
2. 已有数据迁移模式
3. 一键回滚
4. 健康监控

## M1：基础内核 + 首次引导模式

### 范围
1. 冻结应用画像与兼容性分级（Supported / Experimental / Blocked）。
2. 建立元数据模型（relocation record、operation log、health snapshot）。
3. 完成迁移状态机骨架与错误码规范（预检/复制/校验/切换/后检/清理）。
4. 实现首次引导模式：
   - 首次使用选择外接盘根路径；
   - 对“尚未存在源数据”的画像，直接在外接盘创建目标目录并建立软链接；
   - 失败时回滚到“未迁移”状态。
5. 搭建最小命令接口与页面骨架（列表、确认弹窗、状态页、健康面板壳）。

### 里程碑验收
1. 首次引导流程可闭环，支持至少 1 个 Supported 画像。
2. 首次引导失败不会残留断链或不一致元数据。
3. 应用重启后，元数据与文件系统状态一致。

## M2：已有数据迁移 + 回滚闭环

### 范围
1. 完成已有数据事务化迁移（复制、校验、原子切换）。
2. 实现自动回滚与手动一键回滚。
3. 增加中断恢复机制（进程异常退出、电源中断后状态恢复）。
4. 接入结构化日志与 trace id。
5. 完成迁移确认交互与风险提示文案。

### 里程碑验收
1. 每个 Supported 画像通过至少 20 轮“迁移 + 回滚”。
2. 迁移每一步强制中断后都可恢复到一致状态。
3. 回滚后应用可正常启动，源路径可读写，断链为 0。

## M3：健康监控 + 发布就绪

### 范围
1. 实现健康监控器（周期轮询 + 挂载事件触发）。
2. 完成健康状态模型与告警面板（healthy / degraded / broken）。
3. 增加元数据与文件系统对账任务（防漂移）。
4. 完成权限缺失、磁盘离线、只读磁盘等恢复引导。
5. 完成发布前测试矩阵、已知限制说明、回滚操作手册。

### 里程碑验收
1. 外接盘离线可在一个健康检测周期内触发告警。
2. 重新挂载后可自动恢复健康，或给出明确修复引导。
3. 发布检查项全部通过且可追踪复现。
